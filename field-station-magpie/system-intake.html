AGENT HANDOFF

Start Here

Copy this instruction into any MAGPIE interface conversation before asking it to commit code or media.

Work in the repository: matchzimmerman/matchzimmerman.github.io

Before changing files:
1. Read /AGENTS.md
2. Read /systems/README.md
3. Check /systems/ for a reserved package matching this interface
4. Update the existing package when one matches; do not create a duplicate family
5. Only scaffold a new package when no registered family applies
6. Keep approved conversation renders in the package's /reference/ directory
7. Put executable source in the package's /src/ directory
8. Update that package's manifest.json and provenance
9. Run: python scripts/sync_magpie_systems.py
10. Run: python scripts/build_archive_index.py
11. Commit the package, registries, and rebuilt index

Do not place source code inside archive/executions and do not overwrite another system's state.

RUNTIME CONTROL

Autonomy Modes

Newly recovered work should remain visible before it is trusted to run unattended.

01

Reference Only

Source and approved output are retained. The archive displays the system, but nothing executes.

02

Manual

The generator is registered and tested, but runs only when explicitly invoked by a recovery agent.

03

Scheduled

The recurring archive workflow may select the generator and retain each output as a new iteration.

04

Stateful Scheduled

Each run inherits persistent state, records mutation and lineage, and may generate event logs or anomalies.
